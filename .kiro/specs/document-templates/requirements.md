# Requirements: 文書テンプレート一覧確定

## Overview

ISO/IEC 27001:2022 認証取得に必要な文書テンプレートの種類と範囲を決定する。
フォーク可能なテンプレートとして `/isms` ディレクトリに配置し、組織がカスタマイズして利用できる形式で提供する。

## User Stories

### US-1: テンプレート一覧の確認

**As** ISMS構築担当者
**I want** 必要な文書テンプレートの一覧を確認したい
**So that** 文書化作業の全体像を把握し、計画を立てられる

#### Acceptance Criteria

1. **AC-1.1**: WHEN ユーザーがテンプレート一覧ページにアクセスする THE SYSTEM SHALL ISO 27001:2022 で必須とされる全文書のテンプレート一覧を表示する
2. **AC-1.2**: WHEN ユーザーがテンプレート一覧を閲覧する THE SYSTEM SHALL 各テンプレートの対応箇条番号・管理策番号を表示する
3. **AC-1.3**: WHEN ユーザーがテンプレート一覧を閲覧する THE SYSTEM SHALL 文書の必須/推奨の区分を表示する

### US-2: テンプレートのフォーク利用

**As** Git に精通した組織の担当者
**I want** リポジトリをフォークしてテンプレートを取得したい
**So that** 自社用にカスタマイズして ISMS 文書を作成できる

#### Acceptance Criteria

1. **AC-2.1**: WHEN ユーザーがリポジトリをフォークする THE SYSTEM SHALL `/isms` ディレクトリ配下の全テンプレートが取得される
2. **AC-2.2**: WHEN ユーザーがテンプレートを開く THE SYSTEM SHALL Markdown 形式で編集可能な状態で表示される
3. **AC-2.3**: WHEN ユーザーがテンプレートを編集する THE SYSTEM SHALL プレースホルダー（`{{組織名}}`等）の置換箇所が明確に示されている

### US-3: テンプレートの内容理解

**As** ISMS 初学者
**I want** 各テンプレートの記載例と解説を確認したい
**So that** 何を記載すべきか理解できる

#### Acceptance Criteria

1. **AC-3.1**: WHEN ユーザーがテンプレートを閲覧する THE SYSTEM SHALL 記載例（サンプルテキスト）が含まれている
2. **AC-3.2**: WHEN ユーザーがテンプレートを閲覧する THE SYSTEM SHALL コメントまたは注釈で記載のポイントが説明されている

## Non-Functional Requirements

### NFR-1: 可搬性

- Markdown 形式で提供（プラットフォーム非依存）
- 外部依存なし（画像・スクリプト不要）
- UTF-8 エンコーディング

### NFR-2: カスタマイズ性

- プレースホルダーは `{{変数名}}` 形式で統一
- 組織固有の情報は明確にマーク
- 不要なセクションは削除可能な構造

### NFR-3: 保守性

- 各テンプレートは独立したファイル
- バージョン管理可能
- 差分確認が容易

## Out of Scope

- PDF/Word/Excel 形式のテンプレート（Phase 5 で対応予定）
- 自動入力・生成機能（Phase 4 で対応予定）
- 多言語対応（日本語のみ）
- 認証機関固有の要求事項への対応

## Dependencies

- ISO/IEC 27001:2022 規格の内容理解
- 既存の解説コンテンツ（`/docs/requirements/`, `/docs/controls/`）
