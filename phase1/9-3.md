# 9.3 マネジメントレビュー

## 概要

| 項目 | 内容 |
|------|------|
| 要求事項番号 | 9.3 |
| 要求事項名 | マネジメントレビュー |
| 上位箇条 | 9 パフォーマンス評価 |
| 構成 | 9.3.1 一般、9.3.2 マネジメントレビューへのインプット、9.3.3 マネジメントレビューの結果 |

## 要求事項の目的

トップマネジメントがあらかじめ定めた間隔でISMSをレビューし、その継続的な適切性、妥当性及び有効性を確保する。マネジメントレビューは、ISMSの戦略的方向性を決定し、継続的改善を推進する重要なプロセスである。

## 9.3.1 一般

```mermaid
flowchart TB
    A[トップマネジメント] --> B[あらかじめ定めた間隔で<br/>ISMSをレビュー]
    
    B --> C{レビューの目的}
    
    C --> D[継続的な適切性の確保]
    C --> E[継続的な妥当性の確保]
    C --> F[継続的な有効性の確保]
    
    D --> G[ISMSの改善・維持の決定]
    E --> G
    F --> G
```

| 確認事項 | 意味 | 確認ポイント |
|---------|------|-------------|
| 適切性 | ISMSが組織のニーズに合っているか | 組織の状況変化への対応 |
| 妥当性 | ISMSが目的を達成できるか | リソースの十分性、プロセスの機能 |
| 有効性 | ISMSが意図した結果を出しているか | 目的の達成度、リスク低減効果 |

## 9.3.2 マネジメントレビューへのインプット

### インプット項目の全体像

```mermaid
graph TB
    subgraph "マネジメントレビューインプット"
        A[a 前回レビューからの処置の状況]
        B[b 外部・内部課題の変化]
        C[c 利害関係者のニーズ・期待の変化]
        D[d 情報セキュリティパフォーマンスのフィードバック]
        E[e 利害関係者からのフィードバック]
        F[f リスクアセスメント結果・対応計画の状況]
        G[g 継続的改善の機会]
    end
    
    A --> H[マネジメントレビュー会議]
    B --> H
    C --> H
    D --> H
    E --> H
    F --> H
    G --> H
```

### 各インプット項目の詳細

#### a) 前回マネジメントレビューからの処置の状況

```mermaid
flowchart LR
    A[前回レビューの決定事項] --> B[処置の実施状況]
    B --> C{完了?}
    C -->|Yes| D[完了報告]
    C -->|No| E[進捗報告<br/>遅延理由<br/>対策]
    D --> F[今回レビューへ]
    E --> F
```

#### b) 外部及び内部の課題の変化

| 変化の種類 | 例 |
|-----------|-----|
| 外部課題の変化 | 法規制の改正、脅威環境の変化、技術動向、市場環境 |
| 内部課題の変化 | 組織体制の変更、事業戦略の変更、システム更新 |

#### c) 利害関係者のニーズ及び期待の変化

```mermaid
flowchart TB
    subgraph "利害関係者"
        A[顧客]
        B[規制当局]
        C[株主・投資家]
        D[従業員]
        E[取引先]
    end
    
    subgraph "ニーズ・期待の変化例"
        F[セキュリティ要求の厳格化]
        G[新たな法規制対応]
        H[リスク開示要求]
        I[働き方の変化]
        J[サプライチェーン要求]
    end
    
    A --> F
    B --> G
    C --> H
    D --> I
    E --> J
```

#### d) 情報セキュリティパフォーマンスのフィードバック

```mermaid
graph TB
    subgraph "パフォーマンス指標"
        D1[不適合及び是正処置]
        D2[監視及び測定の結果]
        D3[監査結果]
        D4[情報セキュリティ目的の達成状況]
    end
    
    D1 --> E[傾向分析]
    D2 --> E
    D3 --> E
    D4 --> E
    
    E --> F[パフォーマンス報告]
```

| パフォーマンス項目 | 報告内容例 |
|------------------|-----------|
| 不適合・是正処置 | 発生件数、原因分類、対応状況、傾向 |
| 監視・測定結果 | KPI達成状況、トレンド、異常値 |
| 監査結果 | 監査所見、不適合件数、改善推奨事項 |
| 目的の達成状況 | 各目的の進捗、達成/未達成の理由 |

#### e) 利害関係者からのフィードバック

```mermaid
flowchart LR
    A[顧客からの苦情・要望] --> E[フィードバック集約]
```

## 9.3.3 マネジメントレビューの結果

### アウトプット要件

```mermaid
flowchart TB
    A[マネジメントレビュー] --> B[結果・決定事項]
    
    B --> C[継続的改善の機会に関する決定]
    B --> D[ISMSの変更の必要性に関する決定]
    
    C --> E[改善活動の指示]
    D --> F[ISMS更新の指示]
    
    E --> G[文書化した情報として保持]
    F --> G
```

| アウトプット | 内容例 |
|-------------|--------|
| 継続的改善の機会 | プロセスの効率化、管理策の強化、教育訓練の充実 |
| ISMSの変更の必要性 | 適用範囲の変更、方針の改定、プロセスの見直し |

## マネジメントレビューのプロセス全体

```mermaid
flowchart TB
    subgraph "準備"
        P1[インプット情報の収集]
        P2[報告資料の作成]
        P3[会議のスケジュール調整]
    end
    
    subgraph "実施"
        E1[レビュー会議の開催]
        E2[各インプットの報告]
        E3[議論・検討]
        E4[決定事項の確定]
    end
    
    subgraph "フォローアップ"
        F1[議事録の作成]
        F2[決定事項の周知]
        F3[処置の実施]
        F4[次回レビューでの状況確認]
    end
    
    P1 --> P2 --> P3
    P3 --> E1 --> E2 --> E3 --> E4
    E4 --> F1 --> F2 --> F3 --> F4
    F4 -->|次回| P1
```

## 文書化要求

マネジメントレビューの結果の証拠として、文書化した情報を利用可能にしなければならない。

| 文書化項目 | 内容 |
|-----------|------|
| レビュー議事録 | 出席者、日時、議題、議論内容 |
| インプット報告書 | 各インプット項目の詳細報告 |
| 決定事項記録 | 決定内容、責任者、期限 |
| 処置実施記録 | 決定事項への対応状況 |

## 実施のポイント

1. **トップマネジメントの参加**: 意思決定権限者の出席確保
2. **定期的な実施**: 少なくとも年1回、状況に応じて追加実施
3. **インプットの網羅性**: 要求される全項目の報告準備
4. **決定事項の明確化**: 責任者と期限を明確に
5. **フォローアップの徹底**: 決定事項の実施状況を追跡
6. **文書化の確実な実施**: 結果の証拠としての記録保持
